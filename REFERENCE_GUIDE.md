## 2. 권장 개발 아키텍처 (3-Layer)

이 프로젝트는 Antigravity 프레임워크의 의도에 따라 다음 3가지 계층으로 관리됩니다.

1.  **Directive (전략)**: 각 수학 개념(예: 푸리에 변환, 선형대수)별 시각화 SOP 정의
2.  **Orchestration (의정)**: 사용자의 요청에 따라 적절한 시각화 라이브러리와 데이터 모델 결정
3.  **Execution (실행)**: D3.js, Three.js, p5.js 등을 활용한 실제 시각화 코드 실행 및 결과물 생성

## 3. 우선순위 전략: "The Brilliant Style"

분석 결과, **Brilliant.org**의 디자인 시스템과 인터랙션 방식을 벤치마킹하는 것을 최우선으로 제안합니다.

- **시각적 고급화**: 다크 모드 기반의 프리미엄 컬러 팔레트 적용 (`brand-extractor` 활용)
- **인터랙티브 경험**: 단순히 보는 수학이 아닌, 사용자가 변수를 조절하며 '만지는 수학' 구현
- **확장성**: 개별 수학 개념을 독립적인 `execution` 스크립트로 분리하여 지속적인 라이브러리 구축

## 4. 향후 로드맵

1.  **브랜드 분석**: `brand-extractor`를 통한 Brilliant.org 스타일 시각적 요소 추출 (색상, 폰트, 간격 등)
2.  **가이드라인 설정**: 추출된 정보를 `SKILLS/brand-guidelines`에 반영
3.  **MVP 구현**: 첫 번째 수학 주제 선정 및 Brilliant 스타일의 시각화 프로토타입 제작
